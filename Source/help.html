Attachmate API Generator

This document updated 11/21/2005

Thank you for your interest in this code generator. It generates many combinations of API's, programming languages, functions, and output formats to get you started coding your own API projects. Not everything has been thoroughly developed and tested, so please consider yourself a member of the development team and report any errors, failures, or unexpected behavior that you may encounter to the Developer Support Team . 

The APIGen development effort is a fairly large undertaking and, to meet our goals for product completion and quality, help from contributors will be necessary.  Please feel welcome to accept a role as a contributor in one or more of the following ways:

1. Submit a code sample that you've prepared or have taken from a customer example that offers a good basis for API-Generated code. When the API Generator outputs great code, everyone who uses it can.

2. Suggest improvements to the format for code the API Generator currently outputs.  For example, the C++ code now generated for Extra! or Reflection OLE Automation is given in the form of an OnInitDialog function that is to be inserted into a new MFC project.  If you have an idea for a simple format that'll produce a freestanding application, we might change the existing output format with your example or add it as another format option.

3. If you have understanding of coding one of the programming language, API, and terminal type combinations not yet completed and would be willing to add to the product, your efforts will be most welcome.  Adding your expertise in code form requires little more than simple string insertion to existing templates.  A code walkthrough is easily arranged via an email to the Developer Support Team . A table of project progress can be seen in the API Generator's About box that will show both what you can run now and what still requires a contributor's expertise and effort.

4. Test one of the API's that are currently ready.  None of them have been formally tested and can't be recommended for use until they have.

5. Test one of the API's with the purpose of updating the instructions at the top of the output file.

6. Either update this help file to show the current list of API's or begin work on a JavaDoc project that was started, but not implemented.

7. Several of the API's are 99% complete other than they require a very small modification to finish them.  The project status information shown in the About box labels them as EAR, Expert Attention Required.
















APIGen Features

APIGen Features -- Detail

Prompt for parameter values

Checking this checkbox will present input controls allowing you to specify parameter values for the calls that you specify.  This is enabled for only a subset of the API's listed.  Some of the API's, such as HLLAPI32 and EAL, are no longer recommended as the choice for new code and so the effort necessary to provide parameter input UI has instead been spent adding new API's and languages.

When this option is not checked APIGen will declare variables and assign values to them for each function call you generate. The same set of variable names will be used throughout the generated code to assist you in preparing a ready to run application. Currently, this checkbox control will become disabled from the moment the Generate button is pressed until the New Project button is pressed. You are therefore, for the first release of APIGen, required to choose between letting APIGen insert variable values or specify them yourself for the whole project that you are generating.



Output code to existing file

If you want to insert the output generated with APIGen into an existing source file then check this checkbox.

When this option is checked a file-open dialog will be presented to you when you press the Generate button. It will prompt you for the name of the file into which you want to place code. After the name of the file is specified APIGen will insert the newly generated function call to the first occurrence in the file of the string "*APIGEN*". No header or cleanup code will be inserted when APIGen inserts code into an existing file. Besides needed variable declarations and assignments only the function calls you specify are added to the file.

Generate source

Press this button to begin generating source code containing the selected function in the selected API and language.

Unless the checkbox labeled Output code to existing file is checked the API Generator will expand downward so a new view control will be visible which presents a file header followed by your requested code. The file header will provide details about the format of the generated output, tips on how to build or use it, and any information that is relevant towards running the output in the target environment.




Append source

Each time the Append Source button is pressed code for the selected function will be added after what has been prepared up to that point. Any time a variable is used in the appended call that has not yet been declared or initialized it will be added in the declarations section of the output.



Finalize listing

The various languages used can require different ways to close up the output file when you signal that you are done choosing functions. In some cases utility procedures might be added to the end of the file. In others, the functions must be closed to suit the programming language. For instance, the end of a VB 6.0 Sub Main procedure requires an End Sub statement, while the end of a Java module requires the closing braces.



Save to file

When you're ready to write the generated output to a file you'll press this button. You will be prompted with a File\Save dialog to specify the name, but a suggested name will be provided that is composed of the date and appropriate file extension. You cannot append to an existing file, and must specify a unique name to complete this operation.



Feature Detail

Airlines SDK

The Airlines SDK provides a capability for automating server-side operations involving Unisys, IBM, and VT terminal sessions.

The output generated for this API-language combination is a freestanding C++ or Java file that, when compiled, will start a session of the type specified and apply the commands specified while running APIGen.

Airlines SDK - C++

The output generated for this API is a freestanding C++ file that includes the launching of an Airlines SDK session that talks to the Airlines server through the COM bridge.

Airlines SDK - Java

The output generated for this API is a freestanding Java file that includes the launching of an Airlines SDK session that talks to the Airlines server.
 

CASL Macros

CASL Macros provide a language that is specific to the architecture of the 32-bit Extra! product. With CASL you can automate a basic set of communications functions, and also make use of a set of control structures and UNIX-like commands.

CASL (Crosstalk Automation Scripting Language) macros orginated in Crosstalk communications software, were later included in the DCA IRMA products, and then in Extra! when the architectures of the 16-bit IRMA and Extra! products were combined into Extra! Personal Client.

EAL

EAL is implemented for the Windows-native Extra! product and provides Windows operating system features such as multi-threading, more flexible event handling, and Windows UI components. It is commonly called EAL, but the DLL that provides it is named ATMAPI32.DLL. .

EAL - C++

The output generated for this API-language combination is an MFC 6.0 OnInitDialog function that is intended to replace one generated by the Visual Studio 6.0.  When creating a new application it is recommended that you generate an MFC dialog application with Visual Studio's application wizard and replace the OnInitDialog function it generates in the CDialog class file with the one APIGen produces.

EAL - VB 6.0

The output generated for this API-language combination is the body of the OnClick() subroutine that you will create when adding a button to a VB UI form.

EAL - C#

The C# language is not yet generated in this version of APIGen.  It should show up in the next release for use in the Visual Studio .Net IDE.
 

EAL - VB.Net

The VB.Net language is not yet generated in this version of APIGen.  It should show up in the next release for use in the Visual Studio .Net IDE.

EAO for ActiveX

The Enterprise Access Objects for ActiveX provide COM based access to an API set that can interact with 3270, 5250, and VT sessions.  The intent for EAO at the concept stage of its development was a product that would provide the features and functions offered before in the Windows native product, Extra! Personal Client.  The benefits of the EAO approach were 1) improve the relative speed of internal processing (A.K.A. a more efficient architecture compared to EPC); 2) the ability to minimize the emulator's hard disk and memory footprint by providing a means to load only the objects required; and, 3) provide the best proprietary means of distribution, management, and automation of host access products.

The EAO objects were originally developed to perform standalone emulation with the long term intent that they be used as the basis for several other Attachmate products.  The most notable successes in that plan were the use of the EAO for ActiveX code base in the HAS and MCS ActiveX viewers. .  The code bases have diverged, but remain similar.

EAO for ActiveX - C++

The output generated for this API-language combination is an MFC 6.0 OnInitDialog function that is intended to replace one generated by the Visual Studio 6.0.  When creating a new application it is recommended that you generate an MFC dialog application with Visual Studio's application wizard and replace the OnInitDialog function it generates in the CDialog class file with the one APIGen produces.

EHLLAPI

EHLLAPI is Attachmate's best effort at duplicating the API library ACS3EHAP, released by IBM as a feature of its earliest host emulation products to automate 3270 and 5250 hosts.  Even though some newer API's do not include the exact same function list, EHLLAPI's function list represents the basis of all host automation. 

EHLLAPI is implemented for the Windows-native Extra! product.  Extra!'s 32-bit implementation is prepared to perform 3270, 5250, and VT host automation.  Because the function list is the same as given in the original MS/PC-DOS specification, it is somewhat limited to a set of DOS capabilities.  For most needs that is sufficient, but if you are interested in Windows options such as multi-threading or enhanced event/message handling you might choose to use one of the Windows-native API's.

EHLLAPI - C++

The EHLLAPI C++ source code generated by APIGen is in a form that can be compiled as-is in a Visual C++ IDE.   It will link with its function set dynamically by loading EHLAPI32.DLL at runtime.

The directions given in the source file's header block explain how to build the file.  The key point to building the EHLLAPI C++ output is simply creating the workspace. The steps recommended are given here:
1) With Visual C++ 6.0 running choose File\Close Workspace.
2) Open the EHLLAPI C++ file just generated by APIGen.
3) Press F7 to start building the file.  You will first be prompted for whether or not you want Visual C++ to create a new project.  Answer with OK and the module will build and be ready to run.

HLLAPI

Attachmate HLLAPI is an extension of EHLLAPI prepared for use in Attachmate's Windows products. Implemented in the library file HLLAPI32.DLL, it was developed to fill the void left when, for many years IBM did not extend its DOS EHLLAPI specification to implement features that became available with the introduction of Windows.  With HLLAPI32, Attachmate provided customers use of OS features such as multi-threading, more flexible event handling, and Windows UI components.

HLLAPI is implemented for the Windows-native Extra! product.  It features a relatively large number of functions when compared to EHLLAPI, WHLLAPI, IHLLAPI, or PCSHLL32, as each function's signature is customized to the function's needs.  Many HLLAPI functions pass API-proprietary structures for arguments which can better organize data used in host transactions or may hold event handling capabilities.

HLLAPI - C++

The C++ source code generated by APIGen is in a form that can be compiled after it has been inserted into an existing Visual C++ MFC project.   It will link with its function set dynamically by loading HLLAPI32.DLL at runtime.

The directions given in the source file's header block explain how to build the file.  The key points to building and running the HLLAPI32 C++ output are:
1) Prepare a new Visual C++ Dialog workspace: 
a) Choose the Visual C++ 6.0 File\New menu.
b) Click the Projects tab and select the MFC AppWizard item in the leftside treeview.
c) Ensure that the Location edit control points to the folder in which you want to create your new project, then name it in the Project name field, and press OK.
d) Choose Dialog based when prompted for the type of application you would like to create followed by pressing the Finish button.
2) Replace the generated OnInitDialog function with the one generated by APIGen:
a) When the project is created open the file within the project that is given Dlg.cpp as the last seven characters in its name.
b) Open the C++ file that generated by APIGen.  Copy the OnInitDialog function from this file and replace the one in the file you opened above in step 2a.
3) Import the hllapi32.h header file:
a) Insert the line, "#include "hllapi.h", just below the last #include line at the top of the file that you opened in step 2a.
4) Link to the HLLAPI32.lib static import library:
a) Choose the Visual C++ 6.0 Project\Settings menu item, click on the Link tab, and enter hllapi32.lib into the Object/library modules edit field.
5) Press F7 to start building the file.  You will first be prompted for whether or not you want Visual C++ to create a new project.  Answer with OK and the module will build and be ready to run.

IHLLAPI

Attachmate IHLLAPI is an INFOConnect API prepared to support Unisys host automation in a Windows environment. Implemented in the library file IHLAPI32.DLL, it is a port of WHLLAPI32 with some functions inappropriate for the Unisys environment removed, and others added to extend IHLLAPI beyond what was available in WHLLAPI32.

If you are going to code with this API you may need to obtain the keycode for the IHLLAPI Programmer's Guide, as it was encrypted in some INFOConnect releases. The keycode for the file, Api20ref.pdf, is 99-4126798.

The directions given in the source file's header block explain how to build the file.  The key points to building and running the OLE Automation C++ output are:
1) Prepare a new Visual C++ Dialog workspace: 
a) Choose the Visual C++ 6.0 File\New menu.
b) Click the Projects tab and select the MFC AppWizard item in the leftside treeview.
c) Ensure that the Location edit control points to the folder in which you want to create your new project, then name it in the Project name field, and press OK.
d) Choose Dialog based when prompted for the type of application you would like to create followed by pressing the Finish button.
2) Copy the file, extra.tlb, from your Extra! install folder to the folder in which your new project resides.
3) Replace the generated OnInitDialog function with the one generated by APIGen:
a) When the project is created open the file within the project that is given Dlg.cpp as the last seven characters in its name.
b) Open the C++ file that generated by APIGen.  Copy the OnInitDialog function from this file and replace the one in the file you opened above in step 2a.
4) Import the Extra! type library file:
a) Insert the line,
#import "extra.tlb" named_guids
 
just below the last #include line at the top of the file that you opened in step 2a.
5) Declare the Extra! COM objects: 
a) Insert the lines that follow just below the line that you inserted in step 3a.
EXTRA::_SystemPtr poSystem;
EXTRA::SessionsPtr poSessions;
EXTRA::_SessionPtr poSession;
EXTRA::ScreenPtr poScreen;
EXTRA::OIAPtr poOIA;
EXTRA::AreaPtr poArea;
EXTRA::_ScreenWaitPtr poScreenWait;
EXTRA::WaitsPtr poWaits;
 
6) Ensure that the name of the class name before the OnInitDialog function's scope resolution operator (::) is accurate.  It is most likely that the file you generated had a class name
of COLE32CPPDlg.  You will need to change that class to the name you chose for the project above in step 1c.
7) Press F7 to start building the file.  You will first be prompted for whether or not you want Visual C++ to create a new project.  Answer with OK and the module will build and be ready to run.

PCSHLL32

PCSHLL32 is the Attachmate implementation of the specification underlying version 4.1 of IBM's PC3270 and PC400 API's.  IBM's API was introduced in version 4.1 in 1995 and updated to version 4.3 in 2002.  Attachmate's EHLLAPI and PCSHLL32 have not been updated to match the 4.3 specification.

This is IBM's latecoming implementation of a Windows API for IBM host automation prepared for use with Extra!.  It provides nothing beyond what was provided years before in EAL, Attachmate HLLAPI, and WinHLLAPI except the ability to specify as many as four characters in session short names.  As mentioned PDA #72877, Extra!'s Session Manager cannot handle the four character short names, so there is no compelling reason to use PCSHLL32 unless you are involved in migrating a customer from a competing product. 

One other issue that exists in PCSHLL32 and is documented in PDA #171554 concerns the fact that any time Extra!'s API trace utility is recording and a warning condition is encountered, a memory access violation will occur which blasts the PCSHLL32 client application.

PCSHLL32 - C++

The PCSHLL32 C++ source code generated by APIGen is in a form that can be compiled as-is in a Visual C++ IDE.   It will link with its function set dynamically by loading PCSHLL32.DLL at runtime.

The directions given in the source file's header block explain how to build the file.  The key point in building PCSHLL32 C++ output is creating the workspace.  The steps recommended are given here:
1) With Visual C++ 6.0 running choose File\Close Workspace.
2) Open the PCSHLL32 C++ file just generated by APIGen.
3) Press F7 to start building the file.  You will first be prompted for whether or not you want Visual C++ to create a new project.  Answer with OK and the module will build and be ready to run.
